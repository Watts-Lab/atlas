# Guide Overview

Atlas turns scientific papers into structured, review-ready data. A typical workflow starts with a project, adds a set of papers, defines the fields you want extracted, and then reviews the resulting table.

If you are new to the system, start with [Concepts](./concepts.md). Atlas uses a small vocabulary consistently across the web app, API, and MCP tools.

## Typical Workflow

1. Create a project.
2. Define or select the features you want to extract.
3. Upload papers.
4. Wait for processing to finish.
5. Review and export results.
6. Adjust features or prompts and reprocess when needed.

Start with [Get Started](./get-started.md) for the shortest path through the application.
